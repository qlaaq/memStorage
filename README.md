### Инструкция по использованию библиотеки `memStorage`


#### 1. Установка через CDN (unpkg)

Для подключения библиотеки добавьте в HTML‑документ тег `<script>` со ссылкой на unpkg:

```html
<script src="https://cdn.jsdelivr.net/gh/qlaaq/memStorage/dist/memStorage.min.js"></script>
```

**Размещение тега:**
- В секции `<head>` — если нужен доступ к `mem` сразу при загрузке страницы.
- Перед `</body>` — если библиотека требуется после загрузки контента.


**Параметры URL:**
- `memStorage@latest` — загрузка последней версии. Для фиксации версии укажите номер (например, `memStorage@1.0.0`).
- `/dist/memStorage.min.js` — путь к оптимизированному файлу.

#### 2. Проверка подключения

После загрузки страницы откройте консоль браузера (F12 → Console) и введите:
```javascript
mem
```
Если библиотека подключена, отобразится объект `Proxy`.


#### 3. Основные операции

**3.1. Сохранение данных**  
Присваивайте значения свойствам `mem`. Данные автоматически сохраняются в `localStorage` в формате JSON.

```javascript
mem.user = { name: "Алексей", age: 25 };
mem.count = 10;
mem.tags = ["js", "storage"];
```

**3.2. Чтение данных**  
Обращайтесь к свойствам `mem` как к обычным переменным. Данные загружаются из `localStorage` и десериализуются.

```javascript
console.log(mem.user.name);     // "Алексей"
console.log(mem.count);          // 10
console.log(mem.tags[0]);         // "js"
```

**3.3. Удаление данных**  
Используйте оператор `delete` для удаления отдельных ключей или всего хранилища.

```javascript
delete mem.user;    // Удаляет ключ "user"
delete mem.count;    // Удаляет ключ "count"
delete mem.__all;    // Очищает всё localStorage
```

#### 4. Особенности работы

- **Автосериализация**: все значения преобразуются в JSON при сохранении.
- **Обработка ошибок**: при проблемах с `localStorage` (например, превышение квоты) выводится сообщение в консоль.
- **Возврат значений**:
  - Если ключ не найден (`null`), возвращается `undefined`.
  - Если сохранённое значение не является JSON, оно возвращается как строка.
- **Защита объекта**:
  - `mem` нельзя перезаписать (`mem = ...` не сработает).
  - `mem` нельзя удалить (`delete window.mem` не сработает).

#### 5. Примеры использования

**Пример 1. Сохранение и загрузка пользовательских настроек**
```javascript
// Сохранение
mem.settings = {
  theme: "dark",
  fontSize: 14
};

// Загрузка
const theme = mem.settings.theme;
console.log(`Тема: ${theme}`); // "Тема: dark"
```

**Пример 2. Работа с счётчиком**
```javascript
// Инициализация
mem.counter = 0;

// Увеличение
mem.counter += 1;

// Вывод
console.log(mem.counter); // 1
```

**Пример 3. Управление списком задач**
```javascript
// Создание списка
mem.tasks = [
  { id: 1, title: "Купить хлеб", done: false },
  { id: 2, title: "Позвонить маме", done: true }
];

// Добавление задачи
mem.tasks.push({ id: 3, title: "Сдать отчёт", done: false });

// Удаление задачи
mem.tasks = mem.tasks.filter(task => task.id !== 2);
```

#### 6. Возможные проблемы и решения


- **Ошибка 404 (файл не найден)**  
  Проверьте правильность URL в теге `<script>`. Убедитесь, что версия библиотеки существует.

- **Библиотека не работает**  
  Убедитесь, что тег `<script>` расположен **до** кода, использующего `mem`.


- **Проблемы с CORS**  
  Если сайт и CDN находятся на разных доменах, проверьте настройки CORS на сервере.


- **Некорректная работа после обновления**  
  Замените `latest` в URL на конкретную версию (например, `@1.0.0`), чтобы избежать несовместимостей.


#### 7. Локальная установка (альтернативный вариант)


Скачайте файл `memStorage.min.js` и подключите его локально:
```html
<script src="/путь/к/memStorage.min.js"></script>
```
Замените `/путь/к/` на актуальный путь к файлу на вашем сервере.


#### 8. Обновление библиотеки


Для перехода на новую версию измените номер версии в URL:
```html
<script src="https://cdn.jsdelivr.net/gh/qlaaq/memStorage@v1.0.0/dist/memStorage.min.js"></script>
```

#### Заключение

Библиотека `memStorage` упрощает работу с `localStorage`, предоставляя интуитивно понятный интерфейс через объект `mem`. Вы можете:
- Сохранять, читать и удалять данные с помощью простых операций.
- Автоматически обрабатывать сериализацию/десериализацию JSON.
- Защищать хранилище от случайного удаления (`__all`).
